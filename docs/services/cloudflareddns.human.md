## What this service actually does

Updates Cloudflare DNS A/AAAA records to match the public egress IP of `tmtdockp04`. The fleet relies on this to keep `*.themillertribe.org` and `*.themillertribe-int.org` resolvable to the Dell R640's current WAN address.

Other services (Caddy, external clients) use the DNS records generated by this service for routing. It does not handle traffic; it only maintains the mapping. It is NOT a Cloudflare Tunnel daemon despite the CMDB `service_type` classification. It does not store logs or state locally beyond container logs.

### Configuration shape

Environment variables drive all behavior. `INTERVAL=300` forces a check every 5 minutes. `CF_HOSTS` and `CF_ZONES` define the scope. `DETECTION_MODE=dig-whoami.cloudflare` specifies the public IP source. No YAML config files are mounted or parsed.

## Dependencies

### Hard (service cannot start or operate without these)
- **Internet Connectivity** — Required for `one.one.one.one` healthcheck and Cloudflare API calls.
- **Cloudflare API** — Requires valid `CF_USER` and `CF_APIKEY`.
- **Host Egress IP** — Tied to `tmtdockp04`'s public IP.

### Soft (degrade if these are down, but service still functional)
- None.

### Reverse (services that depend on THIS service)
- **Caddy** — Depends on DNS resolution for external access. If IP changes and this service is down, external access fails until manual fix.
- **External Clients** — Any device outside LAN relying on domain names.
